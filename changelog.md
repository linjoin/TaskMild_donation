
# TaskMild v2.0.0 更新日志

**发布日期**: 2026-04-18
**版本代号**: v2.0.0

---

###  核心更新概览

本次更新重构了核心监控模块，修复了导致监控失效的严重底层 Bug，并引入了全新的 Web 管理界面。重点优化了功耗控制，确保在非活动状态下 **零 CPU 占用**。

---

###  新增功能

1.  **Web 管理界面(未完善)**
    *   新增基于 Axum 的异步 Web 服务器，默认端口 `8080`。
    *   实时数据看板：显示前台应用、屏幕状态、CPU/内存占用及累计释放内存。
    *   进程管理：支持查看进程列表、搜索、排序及手动查杀。
    *   配置热更新：支持通过 Web 端修改配置文件并热加载生效。
    *   日志流：基于 SSE (Server-Sent Events) 的实时日志推送。

2.  **全局状态统计**
    *   引入原子计数器 `TOTAL_FREED_KB`，跨线程高效统计累计释放内存。
    *   提供 `get_global_stats` 接口，解决 Web 层与核心层的数据同步问题。

---

###  重要修复

1.  **Netlink 屏幕监控修复 (致命)**
    *   **问题**：旧版使用 `String` 处理 Netlink 消息，`\0` 分隔符导致字符串截断，`FB_EVENT_BLANK` 等关键事件匹配失败。
    *   **修复**：重构为字节切片 (`&[u8]`) 匹配逻辑，确保底层事件精准捕获，屏幕状态监控现已完全生效。

2.  **背光路径检测修复 (高危)**
    *   **问题**：硬编码 `/sys/class/backlight/panel0-backlight` 导致绝大多数设备无法读取亮度。
    *   **修复**：实现 `find_brightness_path` 动态扫描机制，自动适配不同厂商的背光设备路径。

3.  **CPU 使用率计算优化**
    *   显式处理 Linux 时钟周期 (Clock Ticks, 通常为 100Hz)，修正高负载下的数值波动。


---

###  性能优化

1.  **低功耗核心**
    *   完善了 `epoll` + `timerfd` 事件循环机制。主线程在无事件时完全休眠，**杜绝空转轮询**。
    *   动态调整扫描间隔：低内存压力下间隔 60s，高压力下缩短至 10s。

2.  **零拷贝解析**
    *   进程信息解析 (`get_process_info`) 继续沿用栈上 Buffer，避免高频 `malloc/free` 开销。

3.  **原子操作**
    *   统计数据更新使用 `AtomicU64`，无锁设计，消除多线程竞争。

---

###  模块与脚本

*   **customize.sh**: 优化网络检测逻辑，增强授权验证的健壮性，支持版本差异检测。
*   **service.sh**: 增加启动重试与 PID 存活检测，确保守护进程的稳定性。

---

###  抨弃清理

*   **代码清理**: 移除所有冗余注释，保持代码库整洁。
*   **逻辑简化**: 移除无效的 `AppState.stats` 缓存，改为实时查询，降低内存占用与状态不一致风险。

---

###  升级建议

v2.0.0 是一次重大更新，解决了 v1.x 版本中监控失效的核心问题。**强烈建议所有用户升级**。
